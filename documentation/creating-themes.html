Creating Themes

This tutorial will provide you with everything you need to know about themes in Respond. This tutorial includes the location of themes, creating a new theme, and a description of all the files that make up a theme.

Sites as Themes

Our goal with Respond is to make building themes as easy as building a site.  In fact, all of our themes can be viewed outside of Respond using a standard browser.  The best way to get started building a theme with Respond is to start with one of our existing themes.  The Bootstrap theme is a great place to start.  You can view it on Github.  

The rest of this document will outline the core parts of the Bootstrap theme so you can understand what goes into making your own theme.

Theme Settings

The settings.json file is found at private/settings.json.  The role of this file is to provide Respond with instructions on how to interact with your theme.  The file is copied into the resources/sites/site-name folder when the site is created using the theme. The settings file has two types of settings.  The first are private settings that are not editable by the end user.  Examples of these are the editable, sortable, blocks, and framework settings below.  These can only be changed by the theme creator or bythe app administrator when the theme is used to create a site.  The second type of setting is a public setting.  These have the type attribute associated with them.  These settings are exposed to the end user in the Settings menu item within Respond.  All settings can change aspects of the themes and can also be used within plugins.  See the table below for more details on how to use these settings.

[
    {
        "id": "editable",
        "value": "[role=main]"
    },
    {
        "id": "sortable",
        "value": ".col, .column"
    },
    {
        "id": "blocks",
        "value": ".row"
    },
    {
        "id": "framework",
        "value": "bootstrap"
    },
    {
        "id": "grid",
        "value": [
          {
            "name": "1 Column",
            "desc": "100%",
            "html": "<div class=\"block row\"><div class=\"col col-md-12\"></div></div>"
          },
          {
            "name": "2 Column",
            "desc": "50%/50%",
            "html": "<div class=\"block row\"><div class=\"col col-md-6\"></div><div class=\"col col-md-6\"></div></div>"
          }
        ]
    },
    {
        "id": "theme-logo",
        "label": "Logo",
        "description": "Primary logo of the site",
        "type": "image",
        "value": "resources\/logo.png",
        "sets": "attribute",
        "attribute": "src"
    },
    {
        "id": "menu-background-color",
        "label": "Menu Background Color",
        "description": "Change the menu background color of the site",
        "type": "color",
        "value": "#B6C3CF",
        "sets": "css",
        "css": "nav{background-color: config(--menu-background-color);} body.home main .jumbotron p{background-color: config(--menu-background-color);}"
    }
]
Setting Description
editable A comma separated list of selectors to elements that are editable
sortable A comma separated list of selectors to elements that are sortable. These should be a container of relative placed elements.
blocks The CSS selector for blocks in the theme. Blocks are containers for other elements (e.g. .row in Bootstrap). They can be sorted with siblings that match their selector.
framework The front-end framework used to render the site (bootstrap, foundation, mdl, custom). Defaults to bootstrap.
grid Defines what a block looks like for your theme. When you click add in the block context menu, the menu contains the options defined in the grid.
[sets=attribute] The theme-logo setting above is an example of the [sets=attribute] setting. This setting can be used to set an attribute of an HTML element for pages in your site. This is used to set a global image across your site (e.g. a logo or icon).
[sets=css] The menu-background-color setting above is an example of the [sets=css] setting. This setting can be used to set css for your site. The css property is copied into a style block on each page of your site. The config(--menu-background-color) is replaced with the value set in the object. The --menu-background-color must be set to the id of the object.

Templates

Templates hold the content that is used to create new pages and provide common headers and footers across pages. You can find the templates in the templates/ folder in your sites directory. Templates have their own unique syntax to inject common settings into your page. To change page settings, you use the settings in the page settings modal.

Setting Description
{{page.title}} The title of the page.
{{page.description}} The description of the page.
{{page.keywords}} The keywords of the page.
{{page.tags}} The tags of the page.
{{page.callout}} A short phrase to call attention to a page.
{{page.url}} The url of the page (not editable in page settings).
{{page.fullUrl}} The full url of the page (not editable in page settings).
{{page.photoUrl}} The url of the selected photo for the page.
{{page.fullPhotoUrl}} The full url of the selected photo for the page.
{{page.thumbUrl}} The thumb url of the selected photo for the page.
{{page.fullThumbUrl}} The thumb url of the selected photo for the page.
{{page.language}} The selected language for the page.
{{page.direction}} The text direction (ltr, rtl) for the page..
{{page.customHeader}} Custom HTML for the header of the page.
{{page.customFooter}} Custom HTML for the footer of the page.

Theme Best Practices

BASE element

Respond adds all links and images relative to the root of the site. As a result, pages in the theme must specify a base element relative to the root. For instance, pages in a /page/ subdirectory must have a base of <base href="../">.

Sortable ellements must be positioned static or relative. These elements cannot be floated.

Immediate children of sortable containers (see settings) must be positioned relative or static. They cannot be floated or positioned absolute. However, children of those children do not have that restriction.

Specify a [role="main"]

One element on the page should be given a role="main" attribute. Respond uses this as the main content of the page. This main content is searchable and added to the pages.json file so that it can be made available to plugins.

Installing a theme

Simply place the theme folder in resources/themes.
